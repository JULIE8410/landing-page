## <Highlight color='#1C5AEC'>예</Highlight>

#### 수직

Orientation prop을 사용하여 수직 메뉴를 만들 수 있습니다.

```jsx
<NavigationMenu.Root orientation="vertical">
  <NavigationMenu.List>
    <NavigationMenu.Item>
      <NavigationMenu.Trigger>Item one</NavigationMenu.Trigger>
      <NavigationMenu.Content>Item one content</NavigationMenu.Content>
    </NavigationMenu.Item>
    <NavigationMenu.Item>
      <NavigationMenu.Trigger>Item two</NavigationMenu.Trigger>
      <NavigationMenu.Content>Item Two content</NavigationMenu.Content>
    </NavigationMenu.Item>
  </NavigationMenu.List>
</NavigationMenu.Root>
```

#### 유연한 레이아웃

`Content`가 렌더링되는 위치에 대한 추가 제어가 필요할 때 `viewport`를 사용합니다.
이는 설계에 조정된 DOM 구조가 필요하거나 `advanced animation`을 구현하기 위해 유연성이 필요할 때 유용합니다. 탭 포커스는 자동으로 유지됩니다.

```jsx
<NavigationMenu.Root>
  <NavigationMenu.List>
    <NavigationMenu.Item>
      <NavigationMenu.Trigger>Item one</NavigationMenu.Trigger>
      <NavigationMenu.Content>Item one content</NavigationMenu.Content>
    </NavigationMenu.Item>
    <NavigationMenu.Item>
      <NavigationMenu.Trigger>Item two</NavigationMenu.Trigger>
      <NavigationMenu.Content>Item two content</NavigationMenu.Content>
    </NavigationMenu.Item>
  </NavigationMenu.List>

  {/* NavigationMenu.Content will be rendered here when active */}
  <NavigationMenu.Viewport />
</NavigationMenu.Root>
```

#### indicator 포함

선택적 `Indicator` 부분을 사용하여 현재 활성 `Trigger`를 강조 표시할 수 있습니다.
이는 `Viewport`와 함께 제공할 화살표 또는 강조 표시와 같은 애니메이션 시각적 큐를 제공하려는 경우에 유용합니다.

```jsx
// index.jsx
import * as NavigationMenu from "@radix-ui/react-navigation-menu";
import "./styles.css";

export default () => (
  <NavigationMenu.Root>
    <NavigationMenu.List>
```


#### submenu 포함

`NavigationMenu`를 중첩하고 `Root` 대신 `Sub` 부분을 사용하여 하위 메뉴를 만듭니다.
Submenu는 `Root` navigation menu와 다르게 작동하며 한 항목이 항상 활성화되어야 한다는 점에서 `Tab` 과 유사하므로 기본값을 할당하고 설정해야 합니다.

```jsx
<NavigationMenu.Root>
  <NavigationMenu.List>
    <NavigationMenu.Item>
      <NavigationMenu.Trigger>Item one</NavigationMenu.Trigger>
      <NavigationMenu.Content>Item one content</NavigationMenu.Content>
    </NavigationMenu.Item>
    <NavigationMenu.Item>
      <NavigationMenu.Trigger>Item two</NavigationMenu.Trigger>
      <NavigationMenu.Content>
        <NavigationMenu.Sub defaultValue="sub1">
          <NavigationMenu.List>
            <NavigationMenu.Item value="sub1">
              <NavigationMenu.Trigger>Sub item one</NavigationMenu.Trigger>
              <NavigationMenu.Content>
                Sub item one content
              </NavigationMenu.Content>
            </NavigationMenu.Item>
            <NavigationMenu.Item value="sub2">
              <NavigationMenu.Trigger>Sub item two</NavigationMenu.Trigger>
              <NavigationMenu.Content>
                Sub item two content
              </NavigationMenu.Content>
            </NavigationMenu.Item>
          </NavigationMenu.List>
        </NavigationMenu.Sub>
      </NavigationMenu.Content>
    </NavigationMenu.Item>
  </NavigationMenu.List>
</NavigationMenu.Root>
```

#### 클라이언트 측 라우팅 사용

라우팅 패키지에서 제공하는 `Link` 구성 요소를 사용해야 하는 경우 사용자 컴포넌트 구성을 통해 `NavigationMenu.Link`로 구성하는 것이 좋습니다.
이렇게 하면 접근성과 일관된 키보드 제어가 유지됩니다. 다음은 Next.js를 사용하는 예입니다:

```jsx
// index.jsx
import { useRouter } from "next/router";
import NextLink from "next/link";
import * as NavigationMenu from "@radix-ui/react-navigation-menu";
import "./styles.css";

const Link = ({ href, ...props }) => {
  const router = useRouter();
  const isActive = router.asPath === href;

  return (
    <NextLink href={href} passHref legacyBehavior>
      <NavigationMenu.Link
        className="NavigationMenuLink"
        active={isActive}
        {...props}
      />
    </NextLink>
  );
};

export default () => (
  <NavigationMenu.Root>
    <NavigationMenu.List>
      <NavigationMenu.Item>
        <Link href="/">Home</Link>
      </NavigationMenu.Item>
      <NavigationMenu.Item>
        <Link href="/about">About</Link>
      </NavigationMenu.Item>
    </NavigationMenu.List>
  </NavigationMenu.Root>
);
```

```css
/* styles.css */
.NavigationMenuLink {
  text-decoration: none;
}
.NavigationMenuLink[data-active] {
  text-decoration: "underline";
}
```

#### 고급 애니메이션

Advanced animation
`--radix-navigation-menu-viewport-[width|height]` 및 `data-motion['from-start'|'to-start'|'from-end'|'to-end'|'to-end']` 속성을 사용하여
진입/퇴출 방향을 기준으로 `viewport` 크기와 컨텐츠 위치를 애니메이션화할 수 있습니다.
`position: absolute;` 과 결합하면 항목 간 이동 시 중복되는 애니메이션 효과를 매끄럽게 만들 수 있습니다.

```jsx
// index.jsx
import * as NavigationMenu from "@radix-ui/react-navigation-menu";
```

## <Highlight color='#1C5AEC'>접근성</Highlight>

[`Navigation role requirements`](https://www.w3.org/TR/wai-aria-1.2/#navigation)을 준수합니다.

### <Highlight color='#1C5AEC'>Menubar의 차이점</Highlight>

`NavigationMenu`를 `menubar`와 혼동해서는 안 되지만,
이 프리미티브는 name `menu`를 공유하여 navigation link 집합을 지칭하지만 WAI-ARIA `menu` 역할을 사용하지 않습니다.
`menu`와 `menubar`는 복합 포커스 관리 및 첫 글자 탐색과 같은 복잡한 기능을 특징으로 하기 때문에
데스크톱 응용 프로그램 창에서 가장 흔히 볼 수 있는 native 운영체제 menu와 같이 동작하기 때문입니다.
이러한 기능은 종종 [`unnecessary for website navigation`](https://github.com/w3c/aria-practices/issues/353)으로 간주되며
최악의 경우 기존 웹 사이트 패턴에 익숙한 사용자를 혼란스럽게 할 수 있습니다.

자세한 내용은 [`W3C  Disclosure Navigation Menu`](https://www.w3.org/WAI/ARIA/apg/patterns/disclosure/examples/disclosure-navigation/) 예제를 참조하십시오

### <Highlight color='#1C5AEC'>링크 사용 및 aria-current</Highlight>

menu 내의 모든 탐색 링크에 대해 `NavigationMenu.Link`를 사용하는 것은 중요합니다.
이는 기본 목록뿐만 아니라 `NavigationMenu.Content`를 통해 렌더링되는 모든 콘텐츠 내에서도 적용됩니다.
이렇게 하면 일관된 키보드 상호 작용과 접근성을 보장할 뿐만 아니라 `aria-current` 및 active style 설정을 위한 `active` prop에 대한 액세스도 가능합니다.
타사 라우팅 구성 요소와의 사용에 대한 자세한 내용은 이 예시를 참조하십시오
